### Session Intent
Fix t08 (delete ambiguity) by adding structural write-restriction via "delete" Router task_type + pre-grounding hint.

### Key Files
- `src/agent.rs` — reasoning tool enum, filter_tools_for_task(), Router tests
- `src/prompts.rs` — system prompt step 8, delete example anti-pattern
- `src/pregrounding.rs` — delete intent pre-grounding hint injection

### Decisions Made
- **"delete" task_type vs. plan-level restriction:** Task_type chosen because plans are generated by LLM (unreliable for Nemotron). Structural tool-gating is deterministic.
- **No step-based safety net for "delete":** Unlike "search" (which opens up at step 1+), "delete" stays restricted permanently. The restriction IS the fix.
- **Pre-grounding hint guarded by keyword check:** Only fires if instruction contains "delete" but NOT "capture"/"distill"/"write"/"create". This prevents the hint from interfering with combined tasks (t03).

### Risks
- **Regression on t03**: If Nemotron classifies "capture + distill + delete" as "delete" instead of "edit", it loses write tools. Mitigated by: task_type description explicitly says "capture, distill = edit" and pre-grounding hint is guarded.
- **Other delete+write tasks**: Any task needing both delete AND write would break under "delete" routing. Mitigated by: clear description distinguishing "delete" (remove only) vs "edit" (write+delete combined).
- **Model doesn't choose "delete" type**: If Nemotron still classifies t08 as "edit", the routing fix has no effect. Mitigated by: explicit examples and description hints.
