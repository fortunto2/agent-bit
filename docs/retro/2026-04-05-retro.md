## Failure Patterns

### Pattern 1: Deploy Spin-Loop (split-main-rs) -- CATASTROPHIC
- **Occurrences:** 14 consecutive deploy iterations
- **Root cause:** `/solo:deploy` skill couldn't produce `<solo:done/>` signal for a Rust CLI project (no deployment artifact — it's not a web app). Deploy stage is meaningless for agent-bit.
- **Wasted:** 14 iterations + MAXITER hit (82 min total)
- **Fix:** Add deploy skip logic for CLI/library projects in `solo-dev.sh` — if project has no `Dockerfile`, `Procfile`, `vercel.json`, or deploy target in Makefile, auto-skip deploy stage.

### Pattern 2: Build Spin-Loop (stabilize-decisions) -- SEVERE
- **Occurrences:** 8 consecutive empty build iterations (7 seconds each)
- **Root cause:** `/solo:build` skill exited immediately with code 130 or empty output (session startup failure). No circuit breaker caught the rapid-fire retries.
- **Wasted:** 8 iterations
- **Fix:** Add circuit breaker to `solo-dev.sh` — if 3 consecutive iterations produce no commit diff + <7s duration, abort with CIRCUIT log.

### Pattern 3: OAuth Token Expired (calibrate-outcome-validator review)
- **Occurrences:** 3 review iterations + 1 rate limit
- **Root cause:** macOS Keychain OAuth token expired mid-pipeline. Claude CLI returned 401, pipeline retried endlessly.
- **Wasted:** 3 iterations + restart overhead
- **Fix:** Detect 401/authentication_error in iter output and pause pipeline with AUTHFAIL log instead of retrying.

### Pattern 4: Rate Limit Restarts
- **Occurrences:** 3 rate limit events
- **Root cause:** API rate limits during heavy pipeline usage. Pipeline handled gracefully with 60s backoff.
- **Wasted:** ~3 min per event (minor)
- **Fix:** Already handled. Could increase backoff to 120s to be safer.

### Pattern 5: Build Signal Failure (fix-t03, fix-t08-delete)
- **Occurrences:** 2-3 wasted build iterations per track before success
- **Root cause:** Build skill occasionally exits without `<solo:done/>` on first attempt (context not loaded, session abort).
- **Wasted:** 4-5 iterations total
- **Fix:** Same circuit breaker as Pattern 2.

## Recommendations

1. **[CRITICAL]** `solo-dev.sh` — Add circuit breaker: abort after 3 consecutive iterations with no commit diff and <10s duration. Would have saved 22 wasted iterations (Pattern 2 + 5).

2. **[CRITICAL]** `solo-dev.sh` — Add deploy-skip logic for CLI/library projects. Check for deploy markers (Dockerfile, vercel.json, fly.toml, deploy target in Makefile). If absent, auto-complete deploy stage. Would have saved 14 iterations (Pattern 1).

3. **[HIGH]** `solo-dev.sh` — Detect OAuth/auth errors (grep for "authentication_error" or "401" in iter output) and halt with AUTHFAIL instead of retrying. Would have saved 3 iterations (Pattern 3).

4. **[HIGH]** Pipeline config — For agent-bit specifically, run with stages `build -> review` (skip deploy entirely). This project has `make preflight` as its "deploy" check, which build/review already cover.

5. **[MEDIUM]** Plan scoping — fix-t23-contact-ambiguity was over-scoped (29% criteria met). Auto-plan should limit acceptance criteria to <10 per track. If >10, split into sub-tracks.

6. **[MEDIUM]** Add `scratch/` observation masking to `/solo:build` skill — large tool outputs during long build iterations waste context.

7. **[LOW]** fix-prompt-regression and confidence-reflection plans had 0 checkboxes in specs — inconsistent format. `/solo:plan` should always generate checkbox-style acceptance criteria.

## Suggested Patches

### Patch 1: solo-dev.sh — Circuit breaker for rapid-fire empty iterations

**What:** Abort stage after 3 consecutive iterations <10s with no new commit
**Why:** Patterns 2+5 caused 22 wasted iterations from session startup failures

```diff
# In main iteration loop, after ITER result:
+ # Circuit breaker: rapid-fire empty iteration detection
+ if [[ "$duration_secs" -lt 10 ]] && [[ "$prev_commit" == "$current_commit" ]]; then
+   rapid_fail_count=$((rapid_fail_count + 1))
+   if [[ "$rapid_fail_count" -ge 3 ]]; then
+     log "CIRCUIT" "3 consecutive empty iterations (<10s, no commit) — aborting stage"
+     break
+   fi
+ else
+   rapid_fail_count=0
+ fi
```

### Patch 2: solo-dev.sh — Deploy skip for non-deployable projects

**What:** Auto-skip deploy stage if no deployment config found
**Why:** Pattern 1 caused 14 wasted iterations trying to "deploy" a Rust CLI agent

```diff
# Before entering deploy stage:
+ # Auto-skip deploy for projects with no deployment target
+ if [[ "$current_stage" == "deploy" ]]; then
+   has_deploy=false
+   for f in Dockerfile vercel.json fly.toml netlify.toml wrangler.toml; do
+     [[ -f "$project_root/$f" ]] && has_deploy=true
+   done
+   grep -q 'deploy' "$project_root/Makefile" 2>/dev/null && has_deploy=true
+   if [[ "$has_deploy" == "false" ]]; then
+     log "SKIP" "No deploy config found — auto-completing deploy stage"
+     touch "$state_dir/deploy"
+     continue
+   fi
+ fi
```

### Patch 3: solo-dev.sh — Auth failure detection

**What:** Halt on authentication errors instead of retrying
**Why:** Pattern 3 wasted 3 iterations on expired OAuth token

```diff
# After saving iter log, before checking result:
+ if grep -q "authentication_error\|OAuth token has expired" "$iter_log"; then
+   log "AUTHFAIL" "Authentication error detected — pausing pipeline. Re-auth and restart."
+   exit 1
+ fi
```
