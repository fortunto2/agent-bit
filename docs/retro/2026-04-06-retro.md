## Failure Patterns

### Pattern 1: Deploy Spin Loop (split-main-rs) — 14 iters
- **Root cause:** Deploy skill exits without `<solo:done/>` on CLI project with no hosting target
- **Wasted:** 14 iterations, all same commit 28dc0b0
- **Fix:** `/deploy` SKILL.md — detect CLI/local in CLAUDE.md, auto-done

### Pattern 2: Build Spin Loop (stabilize-decisions) — 8 iters
- **Root cause:** `/solo:build` exits in <10s without signal. Context degradation or plan parsing failure.
- **Fix:** Pipeline circuit breaker — abort after 3 same-commit no-signal iterations

### Pattern 3: Auth/Rate Limit Crashes — ~6 events
- **Root cause:** OAuth token expiry + CLI exit code 130 with empty output
- **Fix:** Content-based auth detection in `solo-lib.sh:check_circuit_breaker()` (grep for 401/auth errors)

### Pattern 4: Global Timeouts — 3 events (329m, 235m, ~143m builds)
- **Root cause:** No per-stage timeout. harden-t23 ran 5.5h in single iteration.
- **Fix:** Per-stage timeouts: build=90m, deploy=30m, review=30m

### Pattern 5: Ghost Iterations — 2 events
- **Root cause:** fix-verifier-regression and new-tasks-t31-t40 had START lines with no ITER output (crashed/timed out before any work)
- **Impact:** Minor (2 iterations), but invisible to metrics

## Recommendations

1. **[CRITICAL]** Pipeline circuit breaker: abort after 3 consecutive same-SHA no-signal iterations. Would save 22+ iterations (60% of all waste).

2. **[CRITICAL]** Per-stage timeout: build=90m, deploy=30m, review=30m. Would catch the 329m runaway build early.

3. **[HIGH]** Deploy skill local-only detection: CLI projects without hosting targets should get instant `<solo:done/>`. Would eliminate the 14-iter spin loop entirely.

4. **[HIGH]** Auth error content detection: `grep -qiE 'authentication_error|401'` in `solo-lib.sh:check_circuit_breaker()` before fingerprint matching.

5. **[MEDIUM]** Build skill spec checkbox enforcement: auto-mark spec.md criteria as plan tasks complete. 3 tracks shipped with 0% checkboxes.

6. **[MEDIUM]** Observation masking: add `scratch/` convention for long-running builds to prevent context degradation.

7. **[LOW]** Progressive rate limit backoff: 60s -> 120s -> 240s instead of fixed 60s.
