## Failure Patterns

### Pattern 1: OAuth Token Expiry Deploy Spin-Loop (CRITICAL — repeat)

- **Occurrences:** 14 consecutive iterations on split-main-rs deploy + 2 on t08 build = 16 total auth failures
- **Root cause:** OAuth token expired. Pipeline retried without circuit breaker firing. The fingerprint-based circuit breaker (`solo-lib.sh:137`) exists but DID NOT trigger — likely because deploy output varied slightly between iterations (session ID changes, different request IDs in 401 error).
- **Wasted:** 16 iterations (76% of all waste)
- **Fix:** Circuit breaker fingerprint uses last-5-lines md5 — session IDs make each failure "unique". Need content-based auth detection (regex for `authentication_error|OAuth token has expired|401`) BEFORE fingerprint check.

### Pattern 2: Rate Limit + No-Progress Stall (fix-t03)

- **Occurrences:** 3 iterations (2 build failures with same SHA + 1 rate limit)
- **Root cause:** First build iter did work but didn't finish. Iter 2 was near-instant (7 seconds) — likely context was exhausted. Iter 3 hit rate limit, correctly detected and recovered.
- **Wasted:** 3 iterations
- **Fix:** Rate limit detection worked. Stall detection (same SHA + no done) would have caught iter 2 earlier.

### Pattern 3: Stale Plan Auto-Cycling

- **Occurrences:** 1 (agent-boost-7, first pipeline START)
- **Root cause:** Leftover plan from previous session loaded first.
- **Wasted:** 1 iteration
- **Fix:** Pre-flight plan freshness check — if plan.md is 100% `[x]`, archive before starting.

## Recommendations

The fingerprint-based check (`solo-lib.sh:163`) fails because session IDs in error output make each failure "unique". **File:** `solo-lib.sh:check_circuit_breaker()` — add explicit auth-error regex check BEFORE fingerprint matching:
   ```
   if grep -qE 'authentication_error|OAuth token has expired|401.*error' "$OUTFILE"; then
     AUTH_FAILS=$((AUTH_FAILS + 1))
     if [[ $AUTH_FAILS -ge 2 ]]; then log CIRCUIT + break; fi
   fi
   ```
